## What was built

- `earshot/agents/tmux_fallback.py`: `TmuxAgentAdapter`, just another AgentAdapter. The Conductor creates and owns a tmux session (never adopts a stale one), launches the harness CLI inside it (config `command`, defaulting by harness), and kills the session on stop. Prompts go through `send-keys -l --` (literal mode so nothing is interpreted as key names) with `load-buffer`/`paste-buffer` for multiline. Output comes from `capture-pane` with ANSI/OSC/control stripping; completion is the proven output-stability pattern (unchanged capture for 2s after new output appeared); echoed prompt lines are dropped so the user's own words are not read back. Known best-effort limits (quiescence-based completion, TUI redraw residue) are documented in the module docstring.
- Registry activation: setting `tmux_pane` on an agent selects this adapter; the value names the owned tmux session.
