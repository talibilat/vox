## What was built

- `earshot/agents/base.py`: the `AgentAdapter` contract (start owning a child process, `send(prompt)` yielding markdown chunks until the turn completes, multi-turn within one session, `stop()`, `alive`), designed against all three harness surfaces so #9 implements without interface churn. `AgentError` carries speakable failure messages.
- `earshot/agents/opencode.py`: the opencode adapter over `opencode serve`'s HTTP + SSE surface, encoding the P0-02 spike findings (health polling for readiness, explicit model pinning, `session.next.step.ended` finish=stop as the completion signal, in-band error events, stall timeout).
- `earshot/agents/__init__.py`: `create_adapter` factory (claude-code/codex branches raise NotImplementedError pointing at #9) and the Phase 1 `first_agent` selection rule.
- `earshot/loop.py`: `ConversationLoop` connecting `on_transcript` to the adapter and the response stream to `speak_stream`. Failure policy: live-but-failing agent gets "The agent is not responding"; a dead agent gets one automatic restart (announced, including the session loss) and a retry; nothing ever raises into the input thread.
- `earshot/daemon.py`: on start with a wake model configured, spawns the configured agent, wires input -> loop -> output; on stop, shuts the agent down.
- `earshot/config.py`: `agents.<name>.model` field added ("provider/model-id"), because the spike showed unpinned sessions can resolve to unauthorized providers.
- `tests/fake_opencode_serve.py`: a scriptable stand-in for `opencode serve` implementing the exact API surface the adapter uses; prompt text drives every failure path (in-band error, stall, process death mid-turn).
- Tests: 25 new (102 total passing). Adapter suite runs against the fake serve as a real spawned child process; loop suite uses scripted adapters; `tests/test_voice_loop.py` drives wake-word fixture audio through InputPipeline -> ConversationLoop -> the fake-served adapter -> OutputPipeline, proving the full voice chain; `TestRealOpencode` runs a two-turn conversation against the real binary and asserts multi-turn recall.

## Manual voice smoke checklist (needs a human with a microphone)

1. `earshot start` with a config pointing at the committed wake model and an opencode agent.
2. Say "Hey Earshot", then an instruction ("list the files in this directory and tell me what you see").
3. Expect the agent's response spoken back with no markdown symbols voiced.
4. Say "Hey Earshot", then a follow-up referencing the first answer; expect session continuity.
5. `kill -9` the opencode process; speak again; expect "The agent stopped. Restarting it now." followed by a fresh-session retry.
