## What was built

- `earshot/conductor/addressing.py`: leading-name extraction with fuzzy matching against the configured names. Similarity blends normalized-text ratio with a consonant-skeleton ratio so vowel-level mishearings ("marvun", "olivea", "sabastian") still route. Thresholds are corpus-driven: at or above 0.80 routes, 0.60 to 0.80 asks aloud, below is unaddressed.
- `earshot/conductor/router.py`: utterance classification and dispatch. Fleet phrases ("agent status" and friends) are classified first and can never reach an agent as prompt text (#13 owns what status says; the default speaks registry statuses). Read requests ("what's your response", "what did you say", "read that again") go to a `read_response` callback seam that #13 implements; the default politely declines. A confidently named command routes through the registry and makes that agent active; unaddressed follow-ups go to the active agent; an ambiguous match speaks "Did you mean <name>?" and holds the command, a spoken confirmation routes it, and anything else falls through to normal routing. Addressing a dead agent gets "not running" instead of a prompt. A bare name switches the active agent.
- Active-agent model: switching is sticky across turns, and the router keeps `Fleet.set_active` updated so the supervisor's restart exemption follows the conversation (turn recovery belongs to the per-agent ConversationLoop).
- `earshot/barge/interrupt.py`: the voice loop now takes any transcript handler exposing handle_transcript/say (a raw AgentAdapter is still accepted and auto-wrapped, so existing call sites did not churn).
- `earshot/daemon.py`: transcripts now flow into the Router; the single-agent case is a one-name fleet where every utterance falls through to the only agent, so Phase 1 behavior is unchanged with one code path.
