## What was built

- `earshot/stt/api_openai.py`: `ApiSttBackend` against an OpenAI-compatible `/audio/transcriptions` endpoint. Audio is wrapped as in-memory WAV and uploaded as multipart form data with a Bearer key read from the environment variable named in config (never a literal key in YAML). All failure classes (HTTP error with status and body detail, unreachable host, timeout, malformed response) raise `BackendUnavailable` with an actionable message.
- `earshot/tts/api_openai.py`: `ApiTtsBackend` against `/audio/speech`, requesting `response_format: "pcm"` (24kHz mono int16 per the OpenAI contract) and streaming the response body chunk-by-chunk into the existing playback path with int16 alignment carry, so first audio does not wait for full synthesis.
- Optional graceful degradation: `stt.api.fallback_to_local` and `tts.api.fallback_to_local` config flags. On API failure the backend transparently delegates to a lazily created local backend (whisper/piper only load if ever needed); TTS fallback audio is resampled to the advertised API rate because the playback stream is already open at that rate.
- Factories in `earshot/stt/__init__.py` and `earshot/tts/__init__.py` now build the API backends from config; local remains the default and the local path never reads any API key or opens a socket.
- Voice-loop hardening: an STT failure during a turn now produces spoken feedback ("I could not transcribe that") instead of crashing the daemon, via a public `ConversationLoop.say`.
- `config.example.yaml` regenerated with the new commented API-mode knobs.
- scipy added as an explicit dependency (it was already in the tree transitively via openwakeword; the TTS fallback resampler uses it directly).

## Manual checklist for a real provider (needs a real API key)

1. `export OPENAI_API_KEY=...`, set `stt.backend: api` and/or `tts.backend: api`.
2. `earshot start`; speak a command; confirm transcription and/or the hosted voice.
3. Unset the key or drop the network; confirm the spoken failure feedback, and with `fallback_to_local: true` confirm seamless local degradation.
