## Real bug found by the live fleet test

Killing a real codex agent exposed a restart race in the codex adapter: the dead process's reader thread wakes on EOF asynchronously and poisoned the NEW process's first request with its "died" sentinel (observed live as "died before answering initialize", leaving cody2 dead after a supervised restart attempt).
Fix: per-process reader state (each spawned process gets its own response dict and notification queue, handed to its reader at spawn), so a stale EOF can only poison its own generation. Regression test included; the live kill test now recovers in ~230ms.
