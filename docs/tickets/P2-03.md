## What was built

- `earshot/conductor/watchers.py`: one `AgentWatcher` thread per fleet member. Watchers execute every dispatched turn, buffer the full markdown response into a size-capped ring (8 responses deep, 100k characters per response, so 16 chatty agents stay bounded by design), and drive registry status: busy while a turn runs, finished when an unread response waits, idle once it has been read. The anti-audio-soup rule is structural: only the ACTIVE agent's turns are streamed to the speaker as they arrive; every other agent works in silence and is heard only when asked.
- Idle/finished detection rides the adapters' native completion signals (the turn stream ending IS the harness's turn-complete event, per docs/control-plane-verdicts.md); no harness was flagged for the polling fallback, so none was built.
- `earshot/conductor/status.py`: `spoken_status` composes the roll-call ("marvin and olivia have finished; sebastian is still working"), grouping finished/working/idle/not-running with natural verb agreement and omitting empty groups.
- Router integration: the Router gained a `dispatch` seam; with the watcher pool plugged in, routed commands run on watcher threads and the read-request and fleet-status seams from #12 are now real (`latest_response_text`, `status_line`). In watcher mode the fleet supervisor owns restarts for ALL agents including the active one (watchers never restart; the active-agent exemption only applies in the legacy ConversationLoop mode), eliminating the dual-owner restart race by construction.
- Daemon wiring: pool constructed with the fleet, router wired to the pool's seams, active-agent probe connected after the router exists, pool stopped on shutdown.
- `finished` added to the registry status vocabulary.
- Failure isolation: a failed turn buffers a speakable failure message, marks the agent dead or finished by actual process liveness, speaks feedback only if that agent was active, and the watcher thread survives for the next turn.
