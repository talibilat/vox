## What was built

- `earshot/speakable/converter.py`: markdown-to-speakable text over markdown-it-py's token stream. Headings and paragraphs become punctuated sentences, bold/italic unwrap, links speak their text and never the URL, inline code reads its content, bullets read as sentences, numbered lists keep their numbers, tables linearize row by row with comma-separated cells, and fenced code blocks follow the configured mode: summarize (default, "A 3 lines python code block."), skip, or read.
- `earshot/speakable/chunker.py`: incremental sentence chunker (boundary regex plus abbreviation/decimal/initial guards) that yields each sentence the moment it completes, invariant to token size.
- `earshot/tts/`: `base.py` (`TtsBackend.synthesize(text) -> Iterator[int16 chunks]` plus `sample_rate`), `local_piper.py` (Piper per the P0-03 latency verdict, voice auto-downloaded to `~/.local/share/earshot/voices` on first use, speed mapped to inverse length_scale), and `create_backend(config)` with the `api` branch pointing at #10.
- `earshot/audio/playback.py`: `Player` with a worker thread, small-slice writes (~46ms per slice) so `stop_and_flush()` halts almost immediately, drops the queue and pending synthesis, and an idle event for tests and the agent loop; `SounddeviceSink` (lazy import) with `abort()` to drop device-buffered audio. This is the hook barge-in (#7) calls.
- `earshot/output.py`: `OutputPipeline.speak_stream` wiring a streaming converter -> chunker -> TTS -> playback. Paragraph text streams sentence-by-sentence without waiting for newlines (an inline-markup balance check prevents voicing a dangling `**` or backtick when a sentence boundary lands inside markup); only fences and tables buffer until complete, because they cannot be interpreted earlier.

## Decisions

- markdown-it-py (MIT, pure Python) parses; hand-rolling a converter for tables/nested constructs is where "hash hash bold star star" bugs come from.
- The first-audio requirement drove the design: an earlier draft buffered paragraphs until a blank line, and its own test caught that the first sentence never played before the stream ended; the rewrite streams sentences out of open paragraphs.
- Piper only in this ticket; Kokoro stays a possible later voice behind the same interface (P0-03 measured it 6 to 20x slower to first audio).
