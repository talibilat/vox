## What was built

- `earshot/agents/claude_code.py`: Claude Code adapter over `claude --print` stream-json with `--resume` session persistence. One process per turn (the real process model), `--permission-mode acceptEdits` for voice-driven turns, per-line stall guard, thinking blocks and hook noise filtered, `result` event as the completion signal.
- `earshot/agents/codex.py`: codex adapter over `codex app-server` JSON-RPC on stdio. app-server chosen over mcp-server after prototyping both (long-lived process, persistent threads, true `item/agentMessage/delta` streaming, `turn/interrupt` available for future barge-in). Reader thread routes responses to callers and notifications to the turn stream; `turn/completed` is the completion signal.
- `earshot/agents/__init__.py`: registry now maps all three `harness:` values to their adapter classes.
- `docs/control-plane-verdicts.md`: per-harness stability verdicts (all three native surfaces are solid; NO harness needs the tmux fallback #14 at current versions), plus the quirks the Phase 2 Conductor must know.
- Fakes for hermetic tests: `tests/fake_claude_code.py` (per-turn process with on-disk session state, so --resume plumbing is genuinely exercised) and `tests/fake_codex_appserver.py` (JSON-RPC stdio with the observed event grammar), each with prompt-driven error/stall/death modes.
- `tests/test_harness_adapters.py`: one behavioral contract parameterized over all three adapters (spawn/own/stop, multi-chunk streaming, multi-turn persistence, in-band error leaves the session usable, mid-turn death raises not hangs, unlaunchable command) plus real-binary two-turn integration tests for Claude Code (pinned to haiku to be gentle on quota) and codex.
- `tests/test_voice_loop.py`: the three-harness validation matrix; the identical fixture-audio voice chain runs against each harness switching only the config `harness:` field.

## Cross-harness bug found by the matrix

`AgentConfig.model` defaulted to the opencode free-tier model (added by an earlier review round), which leaked an opencode-specific value into every harness; the real codex rejected it with invalid_request_error on the first turn.
Fix: `model: null` now means "the harness's own default", and the opencode adapter pins its spike-mandated default itself.
Config validation and the example template were updated accordingly.
