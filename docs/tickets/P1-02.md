## What was built

- `earshot/audio/`: `capture.py` with the `frames()` source abstraction (`MicSource` over sounddevice with lazy import and a drop-oldest queue; `ArraySource` for tests and replay) and `endpointing.py` with `EndOfSpeechDetector` on openWakeWord's bundled Silero VAD ONNX (no torch dependency): utterance ends after configurable continuous silence following speech, with a max-duration backstop.
- `earshot/wakeword/detector.py`: `WakeWordDetector` wrapping `openwakeword.Model` with the patience rule from the P0-03 spike (N consecutive windows above threshold).
- `earshot/stt/`: `base.py` (`SttBackend.transcribe(audio, sample_rate) -> str`), `local_whisper.py` (faster-whisper, int8 CPU), and `create_backend(config)`; P1-07 later implemented the `api` branch against this interface.
- `earshot/input.py`: `InputPipeline` driving wake -> record -> transcribe -> `on_transcript(text)`, stoppable, source- and backend-injectable.
- `earshot/daemon.py`: starts the pipeline in a thread when `wake_word.model_path` is configured; as of #8, transcripts feed the `ConversationLoop` and streamed agent responses are spoken back.
- `tests/fixtures/`: four committed 16kHz wav fixtures (wake positive, hard negative, command sentence, wake-then-command sequence) synthesized with a macOS voice that the wake model never saw in training.
- `tests/test_input_pipeline.py`: 11 tests covering the wake detector (fires on positive, silent on negative speech and silence), end-of-speech (ends in trailing silence not mid-sentence, no end without speech, max-duration backstop), the STT backend (fixture transcription, sample-rate rejection), and the full pipeline on fixtures (wake-then-command produces the right transcript, no wake means no transcript, model-path requirement).

## Decisions

- End-of-speech VAD uses openWakeWord's bundled Silero ONNX rather than the silero-vad pip package, which would pull in torch as a runtime dependency.
- Tests pin a slightly relaxed wake operating point (0.9 sensitivity, patience 3) with the measured fixture margins documented inline, because the committed feasibility-grade model scores the unseen fixture voice at 0.94-0.95, exactly at its default 0.95 threshold; the tests exist to verify detector logic, and model-quality tuning is #15.
- Voice dependencies are core dependencies (numpy, openwakeword, faster-whisper, sounddevice) because local mode is the product default; tests importorskip so the non-audio suite runs without them.
