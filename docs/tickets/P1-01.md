## Design decisions

- argparse over click to keep the runtime dependency surface at exactly PyYAML.
- Dataclasses with a hand-rolled validator instead of pydantic, same reason; error messages carry the full key path (for example `agents.main.harness: expected one of ...`).
- Unknown config keys are hard errors, so typos surface immediately instead of silently using defaults.
- The daemon child runs `python -m earshot.cli run` (hidden subcommand) rather than double-forking; simpler, portable, and the child owns its PID file so `start` can verify readiness.
